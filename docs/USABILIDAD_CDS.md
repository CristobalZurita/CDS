# Usabilidad CDS — Checklist Operativo

Documento operativo basado en el análisis de usabilidad acordado. Aplica a todo el sitio web.

## Objetivos (CDS)
- Navegación clara hacia agendar y contacto.
- Velocidad de carga sólida en desktop y móvil.
- Conversión en CTA principales.
- Retención y retorno a dashboard (clientes).
- Accesibilidad básica y consistencia visual CDS.

## Métricas mínimas
- Tiempo de carga: LCP, TTFB (PageSpeed).
- Conversión: CTA agendar, CTA contacto, login.
- Uso real: usuarios, sesiones, tiempo promedio.
- Accesibilidad: contraste y navegación por teclado (WAVE).
- SEO base: Search Console.

## Eventos GA4 (propuestos)
- `page_view`
- `hero_cta_appointment_click`
- `hero_cta_login_click`
- `contact_submit_success`
- `appointment_submit_success`
- `dashboard_view`
- `ot_detail_view`

## Checklist por sección
### Home
- CTA principal visible y claro.
- Scroll suave + secciones ordenadas.
- Tiempo de carga aceptable.

### Agendar Cita
- Formulario simple, mensajes claros.
- Confirmación de cita visible.

### Contacto
- Formulario funcional y accesible.
- Error y éxito legibles.

### Dashboard Cliente
- Estado OT visible.
- Actualizaciones y progreso claros.

### Admin
- Contexto fijo (cliente/código/instrumento).
- Flujo OT sin fricción.
